# Техническое задание: Telegram-бот "Алина" (AI-support)

## 1. Общее описание

Telegram-бот "Алина" — персональный менеджер для клиентов компании по сопровождению банкротства ArbitrA (Express-Bankrot). Бот авторизует клиентов по ИНН + телефону, связывает их со сделкой в Bitrix24 CRM и предоставляет AI-чат на базе GPT-4o-mini.

**Источник**: перенос из n8n-воркфлоу `AI-support-main` (ID: `LSBIFSgMvCndBnUl`) в самостоятельный Python-сервис.

### 3.2 State Machine (конечный автомат)

Состояние хранится в Supabase (`bot_sessions.state`), НЕ в Aiogram FSM.

```
/start ──────────────► waiting_inn
                            │
                      ввод ИНН + найден в Bitrix
                            │
                            ▼
                      waiting_phone
                            │
                      телефон совпал
                            │
                            ▼
                       authorized ◄──── /start (сброс)
```

Состояния:
- `waiting_inn` — ожидание ввода 12-значного ИНН
- `waiting_phone` — ИНН найден, ожидание подтверждения телефона
- `authorized` — авторизован, доступ к меню и чату

### 3.3 Middleware (session.py)

На каждый входящий update:
1. Извлечь `chat_id` и `update_id`
2. Вызвать Supabase RPC `get_or_create_session(chat_id, update_id, username, first_name)`
3. Если `is_duplicate = true` → молча отбросить update
4. Иначе → передать `session` в handler через `data["session"]`

## 4. Функциональные требования

### 4.1 Команда `/start`

**Действия:**
1. Сбросить сессию в Supabase: `state=waiting_inn`, обнулить `inn`, `deal_id`, `contact_id`, `contact_name`, `bitrix_phone`, `error_count`
2. Отправить приветственное сообщение
3. Убрать reply-клавиатуру (`ReplyKeyboardRemove`)

**Текст приветствия (HTML):**
```
👋 {firstName}, добро пожаловать!

Я — Алина, ваш персональный менеджер по сопровождению процедуры банкротства в компании ArbitrA.

Введите, пожалуйста, ваш <b>ИНН</b> (12 цифр):
```
Если `firstName` отсутствует → `Добро пожаловать!` (без имени).

### 4.2 Состояние `waiting_inn` — ввод ИНН

**Триггер:** текстовое сообщение, `session.state == "waiting_inn"`

**Алгоритм:**
1. Извлечь из текста все последовательности цифр (regex)
2. Найти первую последовательность ровно из 12 цифр
3. **ИНН найден (12 цифр):**
   - Вызвать Bitrix24 batch API — поиск сделки по ИНН
   - **Сделка найдена:**
     - Обновить сессию: `state=waiting_phone`, сохранить `inn`, `deal_id`, `contact_id`, `contact_name`, `bitrix_phone`
     - Отправить сообщение с кнопкой "Поделиться номером"
   - **Сделка НЕ найдена:**
     - Вызвать AI промпт "ИНН не найден"
     - Отправить AI-ответ
4. **ИНН НЕ найден в тексте:**
   - Вызвать AI промпт "Нет ИНН" с контекстом (текст пользователя, кол-во найденных цифр)
   - Отправить AI-ответ

**Сообщение при найденном ИНН (HTML):**
```
✅ ИНН найден!

Для подтверждения личности нажмите кнопку ниже, чтобы поделиться вашим номером телефона:
```
Reply-клавиатура: `[📱 Поделиться номером телефона]` (request_contact: true)

---

### 4.3 Состояние `waiting_phone` — верификация телефона

#### 4.3.1 Пользователь поделился контактом (ContentType.CONTACT)

**Алгоритм:**
1. Нормализовать оба телефона (TG и Bitrix) → последние 10 цифр
2. **Совпали:**
   - Обновить сделку в Bitrix: `UF_CRM_1768296374 = "Авторизирован"`, `UF_CRM_1768547250879 = timestamp_МСК`
   - Обновить сессию: `state=authorized`, сохранить все данные
   - Отправить приветствие + главное меню (2 сообщения)
3. **НЕ совпали:**
   - Вызвать AI промпт "Телефон не совпал"
   - Отправить AI-ответ

#### 4.3.2 Пользователь написал текст вместо контакта

- Вызвать AI промпт "Ожидание телефона"
- Отправить AI-ответ + reply-клавиатуру с кнопкой "Поделиться номером"

## 6. Утилиты

### 6.1 Нормализация телефона
```
normalize_phone("79991234567") → "9991234567"
normalize_phone("89991234567") → "9991234567"
normalize_phone("+7 (999) 123-45-67") → "9991234567"
```
Логика: убрать все нецифровые символы, взять последние 10 цифр.

### 6.2 Извлечение ИНН
```
extract_inn("Мой ИНН 123456789012") → ("123456789012", 12)
extract_inn("Номер 12345") → (None, 5)
extract_inn("Привет!") → (None, 0)
```
Логика: найти все последовательности цифр в тексте regex `\d+`, вернуть первую из 12 цифр и максимальную длину найденной последовательности.

### 6.3 Московское время
Возвращает текущее время МСК (UTC+3) в формате ISO.

## 7. Обработка ошибок

- **Все внешние HTTP-вызовы** (Supabase, Bitrix, OpenAI) обёрнуты в try/except
- При ошибке AI → отправлять fallback-текст (статический):
  - ИНН не найден: `ИНН не найден. Проверьте правильность и повторите.`
  - Телефон не совпал: `Номер не совпадает. Обратитесь к менеджеру.`
  - Чат Алины: `Извините, не могу ответить сейчас. Обратитесь к менеджеру.`
  - Нет ИНН: `Пожалуйста, введите ваш ИНН — 12 цифр (например: 123456789012).`
  - Ожидание телефона: `Нажмите кнопку 📱 Поделиться номером телефона ниже, чтобы я могла подтвердить вашу личность.`
- Логирование ошибок через `logging` (INFO для штатных событий, ERROR для сбоев)

## 11. Верификация

1. Отправить `/start` → бот должен приветствовать и попросить ИНН
2. Ввести невалидный ИНН (5 цифр) → AI объяснит что нужно 12
3. Ввести ИНН которого нет в Bitrix → AI скажет "не найден"
4. Ввести валидный ИНН из Bitrix → бот попросит телефон
5. Написать текст вместо контакта → AI напомнит нажать кнопку
6. Поделиться контактом с неверным номером → AI скажет что не совпадает
7. Поделиться контактом с верным номером → авторизация + меню
8. Нажать "Чат с Алиной" → раздел чата
9. Написать вопрос → AI ответит как Алина
10. Нажать "Оплата" / "Задачи" / "Документы" → заглушки
11. Нажать "Главное меню" → возврат к меню
12. Отправить `/start` повторно → сброс, начать сначала
