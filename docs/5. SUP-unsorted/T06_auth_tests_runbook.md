# T06 — Runbook: Автотесты авторизации

**Как запустить:** открой новую сессию Claude Code и напиши «прогони T06» или «запусти автотесты авторизации».

**Что делает Claude:** выполняет все тест-сценарии ниже через Telegram MCP + Supabase MCP, сбрасывая состояние сессии между группами тестов.

---

## Конфигурация

| Параметр | Значение |
|----------|----------|
| BOT_CHAT_ID | `7892896070` |
| TEST_CHAT_ID | `394254402` (Константин Жигалов / @lobster_21) |
| TEST_INN | `000000000011` |

---

## Протокол выполнения

### Сброс сессии

Перед каждой группой тестов выполнить SQL через Supabase MCP:

```sql
UPDATE bot_sessions
SET state = 'waiting_inn',
    inn = NULL,
    deal_id = NULL,
    contact_id = NULL,
    contact_name = NULL,
    error_count = 0,
    context_data = NULL
WHERE chat_id = 394254402;
```

### Ожидание ответа

После каждого `send_message` — пауза 8–10 секунд, затем `get_history(limit=3)`.

---

## Тест-сценарии

### Группа A: Базовый флоу (state: WAITING_INN)

**[T1] /start без сессии → бот просит ИНН**

1. Сбросить сессию (SQL выше)
2. `send_message(7892896070, "/start")`
3. Проверить последний ответ бота

✅ PASS: ответ содержит `ИНН`
❌ FAIL: ответ не содержит `ИНН`

---

**[T2] Текст без ИНН, 1-я попытка → ответ БЕЗ @Lobster_21**

*(продолжение после T1 — сессия в WAITING_INN)*

1. `send_message(7892896070, "привет, как дела?")`
2. Проверить последний ответ бота

✅ PASS: ответ НЕ содержит `@Lobster_21`
❌ FAIL: ответ содержит `@Lobster_21`

---

**[T3] Текст без ИНН, 2-я попытка → ответ С @Lobster_21**

*(продолжение после T2 — error_count=1)*

1. `send_message(7892896070, "не понимаю что вводить")`
2. Проверить последний ответ бота

✅ PASS: ответ содержит `@Lobster_21`
❌ FAIL: ответ не содержит `@Lobster_21`

---

### Группа B: ИНН и переход к телефону (state: WAITING_PHONE)

**[T4] Валидный ИНН из Bitrix → бот просит телефон**

1. Сбросить сессию
2. `send_message(7892896070, "000000000011")`
3. Проверить последний ответ бота

✅ PASS: ответ содержит `телефон` или `номер`
❌ FAIL: ответ не содержит ни одного из этих слов

---

**[T8] Вопрос «где найти ИНН?» в WAITING_INN → ответ + напоминание про ИНН**

1. Сбросить сессию
2. `send_message(7892896070, "где найти ИНН?")`
3. Проверить последний ответ бота

✅ PASS: ответ содержит `ИНН`
❌ FAIL: ответ не содержит `ИНН`

---

### Группа C: После авторизации (state: AUTHORIZED)

**[T7] /start при авторизованной сессии → меню, НЕ запрашивает ИНН**

1. Установить сессию в AUTHORIZED через SQL:
```sql
UPDATE bot_sessions
SET state = 'authorized',
    inn = '000000000011',
    contact_name = 'Тест'
WHERE chat_id = 394254402;
```
2. `send_message(7892896070, "/start")`
3. Проверить последний ответ бота

✅ PASS: ответ НЕ содержит `Введите` и не заканчивается запросом ИНН
❌ FAIL: бот просит ввести ИНН

---

### Группа D: Phone mismatch (требует F.contact — ручной тест)

**[T5] Чужой контакт → @Lobster_21** — ручной, F.contact нельзя отправить через MCP

**[T6] Правильный контакт → авторизован** — ручной, F.contact нельзя отправить через MCP

*Эти тесты проверяются вручную: поделиться телефоном через кнопку в Telegram.*
