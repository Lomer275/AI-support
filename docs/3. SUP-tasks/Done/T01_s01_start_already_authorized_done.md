# T01 — /start для уже авторизованного клиента

## Customer-facing инкремент

Клиент, который уже прошёл авторизацию, при повторном нажатии `/start` не теряет свою сессию и сразу видит меню — без необходимости заново вводить ИНН и телефон.

## Цель

Сделать так, чтобы `/start` проверял текущее состояние сессии перед её сбросом: если клиент `AUTHORIZED` — показывать меню, а не запрашивать ИНН повторно.

## Изменения в коде

- `handlers/start.py`:
  - Добавлен параметр `session: dict` в хендлер
  - Добавлена проверка `session.get("state") == SessionState.AUTHORIZED` перед сбросом сессии
  - При `AUTHORIZED`: отправляет "✅ {name}, вы уже авторизованы. Чем могу помочь?" + `main_menu_keyboard()`, затем `return`
  - Неавторизованный путь без изменений

## Как протестировать

1. Установить сессию в `authorized` (через Supabase)
2. Отправить `/start` боту
3. Проверить: ответ "✅ Константин, вы уже авторизованы. Чем могу помочь?" + меню
4. Проверить: `state` в Supabase остался `authorized`

## Критерии приёмки

1. ✅ `/start` при `AUTHORIZED` не сбрасывает сессию
2. ✅ Клиент получает сообщение-подтверждение и видит меню
3. ✅ Сессия в Supabase осталась `authorized` (не сбросилась)
