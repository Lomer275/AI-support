# T10 — База знаний компании (COMPANY_FACTS)

## Customer-facing инкремент

Клиент больше не получает ложных сведений о компании: ИИ никогда не называет несуществующий офис в Ростове-на-Дону и не придумывает контакты — только то, что есть в базе знаний.

## Цель

Добавить в `services/support.py` статическую константу `COMPANY_FACTS` с проверенными фактами об «Экспресс-Банкрот» и инжектировать её в начало system-промпта всех 7 агентов и координатора, чтобы модели не «вспоминали» устаревшие данные из обучающей выборки.

## Изменения в коде

- `services/support.py`:
  - Добавить константу `COMPANY_FACTS` после `FALLBACK_ALINA`
  - Содержимое: официальное название, офисы (Краснодар, Воронеж, Барнаул), явный запрет на офис в Ростове, правило анти-галлюцинации
  - Инжектировать `f"{COMPANY_FACTS}\n\n"` в начало `system` в методах: `_r1_lawyer`, `_r1_manager`, `_r1_sales`, `_r2_lawyer`, `_r2_manager`, `_r2_sales`, `_coordinator`

## Как протестировать

1. Авторизоваться в боте с реальным ИНН
2. Спросить: «Есть ли у вас офис в Ростове?»
3. Проверить: ответ не упоминает офис в Ростове, говорит «удалённая работа» или «уточните у специалиста»
4. Спросить: «Как с вами связаться?»
5. Проверить: ответ не выдумывает телефон/email, которых нет в `COMPANY_FACTS`

## Критерии приёмки

1. Константа `COMPANY_FACTS` присутствует в `services/support.py`
2. Все 7 агентов + координатор имеют `COMPANY_FACTS` в начале system-промпта
3. На вопрос об офисе в Ростове ИИ не подтверждает его существование
4. На вопрос о несуществующих фактах ИИ отвечает «уточните у специалиста», а не выдумывает
