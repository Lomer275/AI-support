# T27_s05_smart_escalation — Умная эскалация: промпт Coordinator

## Customer-facing инкремент

Первый явный сигнал раздражения клиента — «почему не знаете», «за что я вам плачу» — сразу переключает на оператора. Бот не продолжает шаблонные ответы после этого.

## Цель

Обновить промпт Coordinator в `support.py`: расширить список триггеров для `switcher=true`, добавить реакцию на повторный вопрос без «не знаю» и на риторические вопросы раздражения.

## Контекст

Текущий Coordinator триггерит `switcher=true` только на явные запросы («свяжите меня», «жалоба»). В диалоге dialog2 три сигнала раздражения подряд проигнорированы — эскалация случилась только на четвёртый.

⚠️ Конфликт с T28, T29: все трогают `support.py` — координировать.

## Scope

- `services/support.py` — промпт метода `_coordinator()`:
  - Добавить в список триггеров `switcher=true`:
    - Риторические вопросы о качестве: «почему не знаете», «кто здесь работает», «за что я плачу», «почему нет ответа», «когда вы ответите»
    - Прямое недовольство: «буду жаловаться», «это безобразие», «расторгну договор», «хочу расторгнуть»
    - Повторный вопрос: клиент задаёт тот же вопрос второй раз подряд, а AI не ответил по существу в предыдущем ответе
  - Добавить явное правило: если `switcher=true` — ответ должен быть коротким человечным сообщением о переводе, без повторения шаблонной информации о деле
  - Пример ответа при эскалации по раздражению: «Понимаю вас. Сейчас переключу на вашего менеджера — он ответит лично.»

## Out of scope

- Изменение логики `text.py` при `switcher=true` — уже реализовано
- Промпты R1/R2 — T28, T29
- Per-chat lock против race condition — T30

## Технические детали

Coordinator получает на вход: вопрос клиента, мнения R1/R2, историю диалога. Текущая инструкция `switcher=true` описана в промпте метода `_coordinator()` в `support.py`.

Новые триггеры добавляются как явный список в промпт — не как fuzzy-логика. Coordinator сам оценивает по контексту истории, был ли вопрос уже задан без ответа.

Шаблон финального сообщения при эскалации по раздражению:
```
«Понимаю вас. Сейчас переключу на вашего менеджера — он ответит лично.»
```
НЕ добавлять в этот ответ информацию о стадии дела.

## Как протестировать

### 1. Риторический вопрос

1. Авторизоваться, написать «почему не знаете» или «за что я вам плачу»
2. Проверить: `switcher=true` в ответе Coordinator, сессия эскалирована
3. Проверить: ответ клиенту короткий, без информации о стадии дела

### 2. Повторный вопрос

1. Написать вопрос → получить уклончивый ответ
2. Написать тот же вопрос снова
3. Проверить: `switcher=true`, переключение на оператора

### 3. Прямое недовольство

1. Написать «я буду жаловаться»
2. Проверить: немедленная эскалация на первое сообщение

### 4. Регрессия — нормальный диалог не эскалируется

1. Задать 5 обычных вопросов о деле подряд
2. Проверить: `switcher=false` для всех

## Критерии приёмки

1. Риторический вопрос с первого раза → `switcher=true`
2. Повторный вопрос без ответа → `switcher=true`
3. Ответ при эскалации по раздражению — без шаблонной информации о деле
4. Обычные вопросы — `switcher=false` (нет ложных срабатываний)
