# T26_s05_au_in_context — АУ в контексте агентов

## Customer-facing инкремент

Клиент спрашивает «кто мой арбитражный управляющий» — получает ответ с реальным ФИО из базы, а не «уточните у специалиста».

## Цель

Добавить ФИО арбитражного управляющего в маппинг сделок Bitrix → `electronic_case` и в строку контекста для агентов.

## Контекст

Сейчас агенты не знают ФИО АУ — поле отсутствует в `cases_mapper.py` и не попадает в `get_case_context()`. В диалоге dialog2 это привело к ответу «уточните у специалиста» на базовый вопрос.

## Scope

- `services/cases_mapper.py`:
  - Найти поле с ФИО АУ в Bitrix сделке (стандартное поле или `UF_CRM_*` — проверить в Bitrix)
  - Добавить поле в `DEAL_SELECT`
  - Добавить маппинг в `build_case_row()` → поле `arbitration_manager TEXT` в таблице `cases`

- Схема `electronic_case.cases`:
  - Добавить колонку `arbitration_manager TEXT` (SQL: `ALTER TABLE cases ADD COLUMN IF NOT EXISTS arbitration_manager TEXT`)

- `services/electronic_case.py` → `get_case_context()`:
  - Добавить АУ в строку контекста: `"Арбитражный управляющий: {arbitration_manager}"` (пропускать если NULL)

- `scripts/sync_bitrix_to_cases.py`:
  - Убедиться что новое поле синхронизируется при следующем прогоне (если `build_case_row` обновлён — синхронизируется автоматически)

## Out of scope

- Изменение промптов агентов — T28 (поведение при отсутствии данных)
- Синхронизация webhook'ом — `cases_mapper.py` используется и в `webhook_server.py`, обновится автоматически

## Технические детали

В Bitrix стандартного поля для ФИО АУ нет — скорее всего это `UF_CRM_*`. Нужно:
1. Открыть любую сделку в Bitrix REST: `crm.deal.get?ID=<deal_id>` и найти поле с АУ
2. Либо спросить у команды какой `UF_CRM_*` используется для ФИО управляющего

Если поле не заполнено в части сделок — в `get_case_context()` строку пропускать (не писать «нет данных» — это будет обработано в T28).

## Как протестировать

### 1. Поле синхронизируется

1. Прогнать `python scripts/sync_bitrix_to_cases.py --limit 5`
2. Проверить в `electronic_case.cases`: колонка `arbitration_manager` заполнена для сделок где АУ назначен

### 2. АУ попадает в контекст агентов

1. Авторизоваться за клиента с заполненным АУ
2. Написать «кто мой арбитражный управляющий»
3. Проверить: ответ содержит реальное ФИО, не «уточните у специалиста»

### 3. Пустой АУ

1. Авторизоваться за клиента без АУ
2. Написать тот же вопрос
3. Проверить: секция АУ отсутствует в `case_context`, агент честно признаёт отсутствие данных (T28)

## Критерии приёмки

1. Колонка `arbitration_manager` добавлена в `electronic_case.cases`
2. Поле синхронизируется из Bitrix через `cases_mapper.py`
3. `get_case_context()` включает ФИО АУ в строку контекста когда поле заполнено
4. При NULL — строка АУ не добавляется в контекст (не пишет «нет данных»)
