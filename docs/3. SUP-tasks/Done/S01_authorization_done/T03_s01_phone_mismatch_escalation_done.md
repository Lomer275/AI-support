# T03 — Эскалация при несовпадении телефона

## Customer-facing инкремент

Клиент, чей телефон не совпал с данными в Bitrix, сразу получает контакт поддержки @Lobster_21 — не застревает без понимания что делать.

## Изменения в коде

- `services/openai_client.py` — `phone_mismatch()`:
  - В системный промпт добавлена инструкция всегда заканчивать упоминанием `@Lobster_21`
- `handlers/contact.py`:
  - `FALLBACK_PHONE_MISMATCH` обновлён: теперь содержит `@Lobster_21`

## Примечание по тестированию

`F.contact` требует нативного шаринга контакта — не тестируется через MCP. Полный e2e тест включён в T06 (ручной чек-лист).

## Критерии приёмки

1. ✅ Промпт `phone_mismatch()` содержит инструкцию упомянуть `@Lobster_21`
2. ✅ Fallback-строка содержит `@Lobster_21`
3. ✅ Состояние после несовпадения остаётся `WAITING_PHONE` (логика не изменена)
