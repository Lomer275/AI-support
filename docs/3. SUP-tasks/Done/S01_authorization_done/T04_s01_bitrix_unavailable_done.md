# T04 — Обработка недоступности Bitrix

## Customer-facing инкремент

При недоступности Bitrix клиент видит "технические проблемы" вместо вводящего в заблуждение "ИНН не найден". error_count не растёт.

## Изменения в коде

- `services/bitrix.py` — `search_by_inn()`:
  - Убран широкий `try/except` — исключения теперь пробрасываются наружу
  - `None` возвращается только при штатном "не найден" (deals пустые)
- `handlers/text.py`:
  - Добавлена константа `FALLBACK_BITRIX_UNAVAILABLE`
  - Вызов `bitrix.search_by_inn(inn)` обёрнут в `try/except`
  - При исключении → фиксированное сообщение о технических проблемах, `return` (error_count не инкрементируется)

## Критерии приёмки

1. ✅ Bitrix недоступен → "Возникли временные технические проблемы. Попробуйте ещё раз через несколько минут."
2. ✅ `error_count` при сбое не увеличивается (остался 0)
3. ✅ Штатный сценарий "ИНН не найден" не затронут
