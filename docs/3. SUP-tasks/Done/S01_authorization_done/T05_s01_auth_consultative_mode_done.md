# T05 — Консультативный режим во время авторизации

## Customer-facing инкремент

Клиент может спрашивать про ИНН и телефон во время авторизации — Алина отвечает и возвращает к шагу. На вопросы не по теме — деликатно переносит ответ на после авторизации.

## Изменения в коде

- `services/openai_client.py` — `no_inn_in_text()`:
  - Добавлены инструкции где найти ИНН (nalog.ru, госуслуги, ФНС)
  - Явный блок дефлекции: вопросы не по авторизации → "отвечу после входа в кабинет"
- `services/openai_client.py` — `waiting_for_phone()`:
  - Добавлены инструкции зачем нужен телефон и что делать если клиент отказывается
  - Явный блок дефлекции: вопросы не по верификации → "отвечу после входа в кабинет"
- `services/openai_client.py` — `_complete()`:
  - Улучшено логирование: при отсутствии `choices` в ответе логируется статус и тело ответа

## Критерии приёмки

1. ✅ `WAITING_INN`: "где найти ИНН?" → объясняет + просит ввести ИНН
2. ✅ `WAITING_PHONE`: "зачем телефон?" → объясняет безопасность + кнопка
3. ✅ `WAITING_INN`: вопрос не по теме → "отвечу после входа" + просит ИНН
4. ✅ `WAITING_PHONE`: вопрос не по теме → "отвечу после входа" + кнопка телефона
