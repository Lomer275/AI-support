# T06 — Автотесты авторизации через Telegram MCP

## Customer-facing инкремент

Все сценарии авторизации (T01–T05) покрыты автоматическими тестами, которые можно запустить одной командой — регрессии обнаруживаются до деплоя.

## Цель

Написать тестовый скрипт, который использует Telegram MCP для отправки сообщений боту и проверяет ответы по ключевым словам (вариант B: keyword matching).

## Scope

- `tests/test_auth_flow.py` — новый файл:
  - Использует Telegram MCP (`mcp__telegram-mcp__send_message`, `get_history`) для взаимодействия с ботом
  - Проверяет наличие/отсутствие ключевых слов в ответах
  - Покрывает все сценарии из state machine S01

### Тест-сценарии

| # | Действие | Ожидаемое | Ключевое слово |
|---|----------|-----------|----------------|
| 1 | `/start` (без сессии) | Бот просит ИНН | `ИНН` |
| 2 | Текст без ИНН (1-я попытка) | Ответ без поддержки | НЕ содержит `@Lobster_21` |
| 3 | Текст без ИНН (2-я попытка) | Ответ с поддержкой | `@Lobster_21` |
| 4 | Валидный ИНН из Bitrix | Бот просит телефон | `телефон` или `номер` |
| 5 | Чужой контакт (phone mismatch) | Ответ с поддержкой | `@Lobster_21` |
| 6 | Правильный контакт | Авторизован, приветствие | `авторизован` или имя клиента |
| 7 | `/start` после авторизации | Меню, не запрашивает ИНН | НЕ содержит `ИНН` |
| 8 | Вопрос "где найти ИНН?" в `WAITING_INN` | Ответ + напоминание про ввод | `ИНН` |

**Сценарий "Bitrix недоступен" (тест 9)** — только ручной, не автоматизируется (требует подмены env).

## Out of scope

- Автоматизация сценария "Bitrix недоступен" (только ручной тест)
- CI/CD интеграция (в рамках этой задачи)
- Тесты post-auth сценариев (меню, чат Алины)

## Как протестировать

1. Убедиться что бот запущен локально (`docker compose up -d`)
2. Задать в `.env` тестовые данные: реальный ИНН клиента из Bitrix + `TEST_CHAT_ID`
3. Запустить: `python tests/test_auth_flow.py`
4. Все 8 тестов должны пройти с `PASS`

## Критерии приёмки

1. Скрипт запускается без ошибок импорта
2. Все 8 тест-сценариев проходят `PASS` на работающем боте
3. Каждый тест выводит: название сценария, результат (`PASS`/`FAIL`), ключевое слово которое искали
4. При падении теста — понятное сообщение что не совпало
