# T02_usability_test_tech_setup — Техническая подготовка к юзабилити-тестированию

## Customer-facing инкремент

Модератор и наблюдатель могут провести тестовую сессию в полностью настроенной среде: тестовые данные в Bitrix готовы для каждого участника, ЮKassa в тестовом режиме принимает платежи, PodpisLon выдаёт документ для подписания, запись экрана работает.

---

## Scope

- Тестовые данные в Bitrix для каждого из 6 участников (ИНН, сделка, задача с дедлайном)
- ЮKassa переключена в тестовый режим, тестовые платежи проходят
- Оператор предупреждён и будет доступен во время сессий
- Screen sharing с телефона настроен и запись работает (проверено на Android и/или iOS)

## Out of scope

- Подготовка документальных материалов (шпаргалка, таблички) — это T01
- Сам прогон dry run — это T03
- Приглашение участников — это T04

---

## Как протестировать

1. **Bitrix:** зайти под тестовым ИНН участника — бот авторизует, сделка и задача видны
2. **ЮKassa:** попробовать оплатить тестовой картой — платёж проходит, бот подтверждает
3. **PodpisLon:** открыть ссылку на документ — документ открывается, поле для подписи есть
4. **Оператор:** написать в чат поддержки — оператор отвечает в течение 2 минут
5. **Screen sharing:** запустить демонстрацию экрана телефона, нажать несколько кнопок — тапы видны на записи

## Критерии приёмки

- [ ] 6 тестовых клиентов в Bitrix: у каждого ИНН, активная сделка, задача с дедлайном
- [ ] ЮKassa в тестовом режиме: тестовый платёж проходит без ошибок
- [ ] Оператор предупреждён о датах сессий
- [ ] Screen sharing с телефона проверен: запись ведётся, тапы видны (Android) или screen share работает (iOS)
