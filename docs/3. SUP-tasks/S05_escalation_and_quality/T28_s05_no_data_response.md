# T28_s05_no_data_response — Ответ при отсутствии данных

## Customer-facing инкремент

Если данных по вопросу клиента нет в базе — бот честно говорит об этом и предлагает переключить на менеджера. Не заполняет пробел шаблоном и не выдумывает.

## Цель

Обновить промпты R1/R2 в `support.py`: запретить заполнять пробелы в данных шаблонными фразами, добавить явное поведение «нет данных → признать + предложить эскалацию».

## Контекст

В диалоге dialog2 на вопрос «кто мой АУ» агент ответил «уточните у специалиста» — это частичный fallback, но неправильный. Нужно: честно признать, предложить переключение, и если клиент соглашается — эскалировать.

Зависит от T26: АУ должен присутствовать в контексте, чтобы правило «нет данных» можно было проверить на реальном случае.

⚠️ Конфликт с T27, T29: все трогают `support.py` — координировать.

## Scope

- `services/support.py` — промпты `_r1_lawyer()` и `_r1_manager()`:
  - Добавить явное правило: если информации по конкретному вопросу нет в `client_context` — признать это прямо: «Этой информации у меня нет»
  - Запретить: заполнять пробел общими фразами («наша команда контролирует», «мы занимаемся этим»)
  - Запретить: выдумывать данные которых нет в контексте
  - Добавить: если признал отсутствие данных → предложить переключение на менеджера

- `services/support.py` — промпт `_coordinator()`:
  - Добавить триггер: если R1/R2 признали отсутствие данных по вопросу клиента → `switcher=true` при согласии клиента на переключение

- Шаблон ответа при отсутствии данных:
  ```
  «Этой информации у меня нет. Хотите, я переключу вас на вашего менеджера, который сможет ответить лично?»
  ```

## Out of scope

- Промпт Coordinator по триггерам раздражения — T27
- Устранение повторений — T29

## Технические детали

Правило добавляется в системный промпт R1 после блока с задачами агента. Пример формулировки:

```
ПРАВИЛО ПРИ ОТСУТСТВИИ ДАННЫХ:
Если в client_context нет информации для ответа на конкретный вопрос клиента:
- НЕ используй общие фразы («мы занимаемся», «наша команда контролирует»)
- НЕ выдумывай данные
- Явно укажи в своём ответе: «Данных по этому вопросу в системе нет»
- Рекомендуй SWITCHER=true для переключения на живого специалиста
```

Coordinator при получении такого мнения от R1 → формирует ответ клиенту с предложением переключения + `switcher=true` если клиент согласился (или сразу, не ожидая согласия — на усмотрение при тюнинге).

## Как протестировать

### 1. Вопрос без данных в контексте

1. Авторизоваться за клиента, у которого поле АУ пустое
2. Написать «кто мой арбитражный управляющий»
3. Проверить: ответ «Этой информации у меня нет. Хотите переключить на менеджера?»
4. Нет шаблонных фраз о «контроле команды»

### 2. Вопрос с данными — ответ нормальный

1. Написать вопрос о стадии дела (данные есть)
2. Проверить: обычный ответ, нет «нет данных»

### 3. Эскалация при отсутствии данных

1. Написать вопрос без данных
2. Ответить «да» на предложение переключить
3. Проверить: `switcher=true`, эскалация

## Критерии приёмки

1. Вопрос без данных → «Этой информации у меня нет» без шаблонных фраз
2. Ответ содержит предложение переключить на менеджера
3. Вопрос с данными → нормальный ответ (нет ложных срабатываний)
4. В ответах нет фраз «мы контролируем», «наша команда занимается» без конкретики
