# T29_s05_no_repetition — Устранение повторений

## Customer-facing инкремент

Бот не повторяет одну и ту же фразу в каждом ответе. Стадия дела упоминается один раз, а не в 5 из 8 сообщений. Шаблон «вам не нужно ничего делать» исчезает.

## Цель

Передавать последние 3 ответа AI в контекст `_r1_lawyer()` и добавить промпт-запрет на дословное повторение уже сказанного.

## Контекст

В dialog2 фраза «Ваше дело находится на стадии реализации имущества без имущества с 2 апреля 2026 года» встречается в 5 из 8 ответов. История диалога уже хранится в `chat_history` (Supabase) и читается через `SupportSupabaseService`. Нужно извлекать из неё последние ответы AI и передавать в промпт.

⚠️ Конфликт с T27, T28: все трогают `support.py` — координировать.

## Scope

- `services/support.py` — метод `answer()`:
  - После получения `history` из `self._supabase.get_history(chat_id)` — извлечь последние 3 ответа AI:
    ```python
    last_answers = [m["content"] for m in history if m["role"] == "assistant"][-3:]
    ```
  - Передать `last_answers` в `_r1_lawyer()` как дополнительный параметр

- `services/support.py` — промпт `_r1_lawyer()`:
  - Добавить блок «уже сказанное»:
    ```
    УЖЕ СКАЗАНО КЛИЕНТУ (не повторяй дословно):
    {last_answers_text}
    ```
  - Добавить правило: не упоминать стадию дела если она уже была упомянута в последних ответах
  - Убрать шаблон «вам не нужно ничего делать» / «не предпринимайте никаких действий» — заменить на конкретику по вопросу или не добавлять вовсе

## Out of scope

- Изменение Coordinator и R2 — только R1 получает last_answers (R2 получает мнение R1 и итак знает контекст)
- Промпт Coordinator — T27
- Поведение при отсутствии данных — T28

## Технические детали

`history` уже читается внутри `answer()` из `SupportSupabaseService`. Метод `get_history` возвращает список dict `{role, content}`. Дополнительного вызова не нужно.

Формирование блока для промпта:
```python
last_answers = [m["content"] for m in history if m["role"] == "assistant"][-3:]
if last_answers:
    last_answers_text = "\n---\n".join(last_answers)
    already_said_block = f"\nУЖЕ СКАЗАНО КЛИЕНТУ (не повторяй дословно эти утверждения):\n{last_answers_text}\n"
else:
    already_said_block = ""
```

Передавать в `_r1_lawyer()` как параметр `already_said: str = ""` и вставлять в системный промпт перед блоком задач.

## Как протестировать

### 1. Нет повторения стадии

1. Написать 4 сообщения подряд (разные вопросы)
2. Проверить: фраза «ваше дело находится на стадии X» — не более 1-2 раз в первых ответах, в последующих — не повторяется дословно

### 2. Нет шаблона «не нужно ничего делать»

1. Написать вопрос о статусе дела
2. Проверить: ответ не содержит «вам не нужно ничего делать» / «не предпринимайте действий» без конкретики

### 3. last_answers корректно передаются

1. Добавить временный лог `already_said_block` в `_r1_lawyer()`
2. Проверить: после 2+ сообщений блок содержит предыдущие ответы AI

## Критерии приёмки

1. Стадия дела не повторяется дословно в каждом ответе подряд
2. Шаблон «не нужно ничего делать» убран из ответов
3. `last_answers` корректно формируется из `chat_history`
4. При пустой истории — блок пропускается, ошибок нет
