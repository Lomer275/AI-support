## Scope

**Автоматическая проверка:**
- Прогнать `python scripts/quality_run.py --output results_s05.json`
- Сравнить с baseline (результаты до S05)
- Убедиться что метрики accuracy и completeness не упали

**Ручной прогон dialog2-сценария:**

Воспроизвести последовательность из dialog2 с реальным клиентом (тестировщик авторизуется за клиента на стадии реализации имущества):

| # | Сообщение | Ожидаемый результат |
|---|-----------|---------------------|
| 1 | «что по моему делу» | Краткая сводка, без шаблона «не нужно ничего делать» |
| 2 | «какой суд сейчас назначен» | Конкретный ответ с названием суда, стадия НЕ повторяется |
| 3 | «когда суд по завершению» | Честный ответ о неизвестности даты, без выдумки |
| 4 | «был суд, что решили» | Признание отсутствия данных или реальный результат, без подмены |
| 5 | «мне нужно разрешение от ау» | Правильный ответ: АУ может выдать разрешение на ПМ |
| 6 | «пришлите разрешение на снятие наличных» | Корректная инструкция, без ложного запрета |
| 7 | «кто мой арбитражный управляющий» | Реальное ФИО из базы (если заполнено) или «нет данных + предложение менеджера» |
| 8 | «почему не знаете» | Немедленная эскалация на ответственного (T27) |

**Проверка эскалации:**
- После шага 8: чат в Bitrix переведён на ответственного менеджера (T25)
- Оператор пишет ответ в Bitrix → клиент получает в Telegram (T24)
- Клиент отвечает в Telegram → оператор видит в Bitrix (T24)

**Чек-лист проблем из dialog2_analysis.md:**
- [ ] Баг 1: «АУ» распознаётся корректно, ответ правильный
- [ ] Баг 2: АУ присутствует в контексте
- [ ] Баг 3: эскалация на первом сигнале раздражения
- [ ] Баг 4: повтор фразы про стадию — не более 1 раза
- [ ] Баг 5: нет шаблона «не нужно ничего делать»
- [ ] Баг 6: нет трёх ответов одновременно
- [ ] Баг 7: вопрос о результате суда — не игнорируется, признаётся отсутствие данных
- [ ] Баг 8: незначимые суммы не упоминаются некстати
