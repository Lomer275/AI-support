# T01_usability_test_materials — Подготовка материалов для юзабилити-тестирования

## Customer-facing инкремент

Модератор (Костя) и наблюдатель получают полный комплект материалов для проведения 6 сессий юзабилити-тестирования: шпаргалку, таблицу наблюдателя, шаблон дебрифа, протокол прогона и шаблон анализа с HADI-карточками.

---

## Scope

- Шпаргалка модератора (md + csv) — сценарии, инструкции, реплики
- Доработанная табличка наблюдателя v2 (csv) — шаги внутри сценариев, колонка критичности
- Шаблон пост-сессионного дебрифа (md)
- Протокол прогона dry run (md) + чеклист техники
- Шаблон анализа результатов (md) + карточки гипотез в HADI-формате

Все файлы размещаются в: `docs/5. SUP-unsorted/usability_test_1/`

## Out of scope

- Технические настройки Bitrix, ЮKassa, PodpisLon — это T02
- Изменения в коде бота — это S02
- Приглашение респондентов — это T04

## Как протестировать

1. Открыть каждый файл в `usability_test_1/`
2. Проверить шпаргалку модератора:
   - Все 6 сценариев из `test_plan_and_script.md` отражены
   - Есть инструкции по введению, завершению и работе с молчанием
3. Проверить табличку наблюдателя:
   - Шаги расписаны внутри каждого сценария
   - Есть колонка критичности
4. Проверить шаблон дебрифа: структура на 5 минут после каждой сессии
5. Проверить протокол dry run: есть чеклист техники и шаги прогона
6. Проверить шаблон анализа: есть HADI-карточки и сводная таблица

## Критерии приёмки

- [ ] Шпаргалка модератора охватывает все 6 сценариев
- [ ] Табличка наблюдателя v2 доработана (шаги + критичность)
- [ ] Шаблон дебрифа рассчитан на 5 мин, покрывает ключевые вопросы
- [ ] Протокол dry run содержит чеклист техники и критерии готовности
- [ ] Шаблон анализа содержит HADI-карточки и сводную таблицу
- [ ] Все материалы проверены и утверждены Костей
