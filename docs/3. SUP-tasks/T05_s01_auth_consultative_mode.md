# T05 — Консультативный режим во время авторизации

## Customer-facing инкремент

Клиент может задавать вопросы прямо во время авторизации ("где найти ИНН?", "зачем нужен телефон?") и получать полезный ответ — после которого Алина мягко возвращает его к текущему шагу.

## Цель

Обновить промпты `no_inn_in_text` и `waiting_for_phone` так, чтобы GPT различал два случая:
- Клиент вводит данные (но неверно) → объяснить ошибку
- Клиент задаёт вопрос → ответить на вопрос + напомнить про текущий шаг

Новое состояние не вводится — всё через системный промпт.

## Scope

- `services/openai_client.py`:
  - Промпт `no_inn_in_text`: добавить инструкцию — если сообщение содержит вопрос (что такое ИНН, где его найти, как авторизоваться и т.д.), ответить и в конце напомнить "введите ваш ИНН из 12 цифр"
  - Промпт `waiting_for_phone`: аналогично — если вопрос по процедуре, ответить и в конце напомнить "нажмите кнопку «Поделиться телефоном»"
  - Оба промпта должны явно задавать Алине роль консультанта по процессу банкротства, а не только ошибки

## Out of scope

- Ответы на вопросы НЕ по процедуре авторизации (юридические вопросы и т.д.) — в `WAITING_INN` / `WAITING_PHONE` Алина остаётся сфокусированной на авторизации
- Смена состояния при вопросах
- Отдельный промпт для вопросов

## Как протестировать

### В состоянии WAITING_INN

1. Написать: "где найти ИНН?" → Алина объясняет + в конце просит ввести ИНН
2. Написать: "что такое ИНН?" → объяснение + просьба ввести ИНН
3. Написать: "123" (невалидный) → Алина объясняет ошибку (без объяснения "что такое ИНН")

### В состоянии WAITING_PHONE

4. После успешного ИНН написать: "зачем нужен мой телефон?" → объяснение + напоминание про кнопку
5. Написать: "не хочу делиться номером" → Алина объясняет необходимость + напоминание про кнопку

## Критерии приёмки

1. В `WAITING_INN`: вопрос "где найти ИНН" → ответ содержит пояснение и напоминание про ввод ИНН
2. В `WAITING_PHONE`: вопрос "зачем телефон" → ответ содержит пояснение и напоминание про кнопку
3. Невалидный ввод (просто цифры, случайный текст) — поведение без изменений

3. **Обновить спецификацию:** статус T05 → `✅`, обновить ссылку
