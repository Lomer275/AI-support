# T03 — Эскалация при несовпадении телефона

## Customer-facing инкремент

Клиент, чей телефон не совпал с данными в Bitrix, сразу получает конкретный контакт поддержки — не застревает в состоянии `WAITING_PHONE` без понимания что делать.

## Цель

В сообщении при несовпадении телефона всегда включать упоминание `@Lobster_21`, чтобы клиент знал куда обратиться.

## Scope

- `services/openai_client.py`:
  - В промпт `phone_mismatch` добавить инструкцию: в конце ответа всегда указывать `@Lobster_21`
- `handlers/contact.py`:
  - Убедиться что fallback-строка (при `None` от OpenAI) тоже содержит `@Lobster_21`

## Out of scope

- Автоматическое уведомление менеджера при несовпадении
- Смена состояния (клиент остаётся в `WAITING_PHONE`)
- Ограничение на количество попыток телефона

## Как протестировать

1. Пройти до состояния `WAITING_PHONE` (ввести валидный ИНН)
2. Поделиться контактом с другим номером (не тем, что в Bitrix)
3. Проверить: в ответе бота есть `@Lobster_21`
4. Проверить: клиент остаётся в состоянии `WAITING_PHONE` (не сбрасывается к ИНН)
5. Проверить fallback: при отключённом OpenAI бот всё равно отвечает с `@Lobster_21`

## Критерии приёмки

1. При несовпадении телефона ответ всегда содержит `@Lobster_21`
2. Fallback-строка в `handlers/contact.py` содержит `@Lobster_21`
3. Состояние после несовпадения остаётся `WAITING_PHONE`
