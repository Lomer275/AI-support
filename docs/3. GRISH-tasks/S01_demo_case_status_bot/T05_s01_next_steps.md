# T05_s01_next_steps — Интент next_steps

## Customer-facing инкремент

Клиент спрашивает «Что мне делать дальше?» — получает конкретные шаги, соответствующие текущей стадии его дела. Не общие советы, а действия под его реальную ситуацию.

## Цель

Добавить обработку интента `next_steps` в `text.py`. Убедиться, что промпт содержит достаточно данных о стадии дела для точного ответа.

## Контекст

T02 и T03 готовы. T04 уже реализует загрузку `case_context` и вызов `AlevtinaService`. T05 добавляет только ветку `next_steps` — инфраструктура та же. Знание стадий БФЛ уже встроено в системный промпт (Часть 2) в T03.

## Scope

- `demo_bot/handlers/text.py` — добавить ветку `elif result["intent"] == "next_steps"`:
  ```python
  elif result["intent"] == "next_steps":
      await message.answer(result["answer"], parse_mode=ParseMode.HTML)
  ```
  (Логика загрузки `case_context` и вызов `alevtina_svc.handle()` уже реализованы в T04 — T05 только добавляет обработку интента)

- Проверить что `case_context` (из `ElectronicCaseService`) содержит поле стадии дела в читаемом виде — если нет, убедиться что `electronic_case.py` возвращает `stage` в строке контекста

## Технические детали

`AlevtinaService` получает `case_context` с данными дела включая стадию. Часть 2 промпта содержит расшифровку всех стадий `[C4:*]` с действиями клиента. GPT сопоставляет стадию из `case_context` с блоком в промпте и формирует конкретные шаги.

Если стадия не входит в список `STAGE_LABELS` — GPT видит сырой код стадии. Промпт (Часть 2) содержит инструкцию: «Нестандартный статус — рекомендуй связаться с менеджером напрямую».

Ответ `next_steps` — список конкретных действий. GPT форматирует с `<b>` заголовком и нумерованным или маркированным списком. ParseMode.HTML применяется при отправке.

## Как протестировать

### 1. Стандартная стадия

1. Авторизоваться за клиента со стадией `C4:EXECUTING` (назначено заседание)
2. Написать «Что мне нужно сделать?»
3. Проверить:
   - `intent == "next_steps"`
   - Ответ содержит конкретные шаги: оплатить депозит, передать квитанции
   - Нет выдуманных шагов

### 2. Стадия «Реализация имущества»

1. Авторизоваться за клиента со стадией `C4:14/15/18`
2. Написать «Что дальше?»
3. Проверить: ответ соответствует блоку «Реализация имущества» из промпта

### 3. Нестандартная стадия

1. Авторизоваться за клиента с редкой/пустой стадией
2. Написать «Что делать?»
3. Проверить: ответ рекомендует связаться с менеджером, нет выдуманных шагов

## Критерии приёмки

1. Вопрос о следующих шагах возвращает `intent == "next_steps"` и осмысленный ответ
2. Шаги соответствуют реальной стадии дела из `case_context`
3. Нестандартная стадия → рекомендация позвонить менеджеру (не выдуманные шаги)
4. Ответ отправлен с `ParseMode.HTML`
5. Ответ не заканчивается вопросом к клиенту
