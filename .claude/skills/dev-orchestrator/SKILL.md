# Dev-Orchestrator Skill

Координирует параллельное выполнение задач разработки. Анализирует зависимости и запускает субагентов.

---

## Входные данные

Пользователь предоставляет одно из:
- Набор задач (`T17 T18 T19 T20`) для выполнения
- Номер спеки (`S04`) — оркестратор берёт все незакрытые задачи
- Просьбу "выполни всё что можно"

---

## Алгоритм выполнения

### Шаг 1 — Собрать контекст

1. Прочитай `SUP-HANDOFF.md` — найди текущие активные задачи и приоритеты
2. Если указана спека — прочитай `docs/2. SUP-specifications/SNN_*.md`
3. Прочитай файлы всех указанных задач из `docs/3. SUP-tasks/`
4. Прочитай `SUP-architecture.md` — для понимания структуры системы

---

### Шаг 2 — Построить граф зависимостей

Для каждой задачи определи:
- **Зависит от:** какие TNN должны завершиться до начала этой
- **Блокирует:** какие TNN ждут завершения этой
- **Независима:** никаких зависимостей с другими задачами из набора

Визуализируй граф:

```
Граф зависимостей:

T17 ──────────────────┐
                      ├──→ T19 ──→ T21
T18 ──────────────────┘

T20 (независима, можно параллельно)
T22 ──→ T23 (последовательно, T22 блокирует T23)
```

---

### Шаг 3 — Определить волны выполнения

Раздели задачи на **волны** (batches):

```
Волна 1 (параллельно):
  - T17 — <название> (нет зависимостей)
  - T18 — <название> (нет зависимостей)
  - T20 — <название> (независима)

Волна 2 (после завершения волны 1):
  - T19 — <название> (требует T17 + T18)

Волна 3 (после T19):
  - T21 — <название> (требует T19)
```

**Покажи план пользователю и жди подтверждения.**

---

### Шаг 4 — Запустить волну 1

Для каждой независимой задачи в волне — запусти субагент через Agent tool:

```
Для каждой задачи в волне N создай субагент с промптом:

---
Ты — разработчик, реализующий задачу [TNN].

Прочитай файл задачи:
  [путь к файлу задачи]

Также прочитай для контекста:
  - CLAUDE.md (правила проекта)
  - SUP-architecture.md (архитектура)
  - [файлы из Scope задачи]

Выполни задачу полностью:
1. Реализуй все изменения из Scope
2. Убедись что критерии приёмки выполнены
3. Не трогай файлы вне Scope задачи
4. При завершении выведи:
   - Список изменённых файлов
   - Статус каждого критерия приёмки (✅/❌)
   - Что протестировал и результаты
---
```

Запускай все задачи волны **в одном сообщении** (параллельно через несколько Agent tool calls).

---

### Шаг 5 — Мониторинг и обработка результатов

После завершения каждой волны:

1. Собери результаты всех субагентов
2. Для каждой завершённой задачи:
   - Проверь что все критерии приёмки выполнены
   - Отметь задачу как завершённую в таблице прогресса
   - Если есть ошибки — залогируй и реши: повторить/эскалировать

3. Выведи статус волны:

```
Волна 1 завершена:
  ✅ T17 — <название> (все критерии выполнены)
  ✅ T18 — <название> (все критерии выполнены)
  ⚠️ T20 — <название> (критерий 3 не выполнен: ...)
```

4. Если задача провалилась:
   - Спроси пользователя: повторить / пропустить / решить вручную
   - Не блокируй следующую волну из-за независимых задач

---

### Шаг 6 — Запустить следующие волны

После успешного завершения волны N — запусти волну N+1 по тому же алгоритму.

Обновляй таблицу прогресса после каждой волны.

---

### Шаг 7 — Обновить документацию

После завершения всех волн:

1. Обнови статусы задач в спецификации (SNN)
2. Обнови `SUP-HANDOFF.md` — отметь завершённые задачи
3. Добавь записи в `SUP-CHANGELOG.md`

Или предложи пользователю запустить `/accept TNN` для каждой задачи.

---

### Шаг 8 — Финальный отчёт

```
✅ Оркестрация SNN завершена

Выполнено: N/M задач

Результаты:
  ✅ T17 — <название>
  ✅ T18 — <название>
  ✅ T19 — <название>
  ⚠️ T20 — <название> (требует внимания: ...)

Изменённые файлы: [список ключевых файлов]

Следующий шаг:
  - /accept T17, /accept T18... — закрыть задачи
  - или /codereview T19 — проверить реализацию
```

## Режимы работы

### Режим "сделай сам" (autonomous)

Запускает субагентов и выполняет всё автоматически с минимальными паузами.

Активируется словами: "сделай сам", "autonomous", "без вопросов", "выполни всё".

### Режим "координатор" (supervised)

После каждой волны показывает результаты и ждёт подтверждения перед следующей.

Активируется по умолчанию или словами: "с паузами", "показывай результаты".

### Режим "план только" (plan-only)

Только строит граф зависимостей и предлагает план — без запуска агентов.

Активируется словами: "только план", "покажи план", "не запускай".

---

## Важные правила

**Изоляция задач:**
- Каждый субагент работает только с файлами своей задачи
- Конфликты файлов (несколько задач меняют один файл) — запускай последовательно, не параллельно

**Определение конфликтов:**
Перед запуском волны проверь Scope каждой задачи. Если две задачи меняют один и тот же файл — они не могут выполняться параллельно, помести их в разные волны.

**Провал задачи:**
- Не останавливай независимые задачи из-за провала одной
- Явно сообщи пользователю что провалилось и почему

**Без самодеятельности:**
- Субагенты работают строго по Scope задачи
- Если в процессе обнаружена новая проблема — логируй, не фиксируй без спроса

**Документация:**
- Всегда обновляй HANDOFF.md и спеку по итогам
- Не закрывай задачи (don't /accept) без явного подтверждения пользователя — если только пользователь не сказал "autonomous"
